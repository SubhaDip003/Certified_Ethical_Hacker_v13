# 🐝 OWASP Top 10 Application Security Risks – 2021 
🔗Source: [https://owasp.org] <br>
OWASP is an international organization that maintains a list of the top 10 vulnerabilities and flaws of web applications. The latest OWASP top 10 application security risks are as follows. 
1. **A01 – Broken Access Control** →
   This happens when users can access things they shouldn’t, like someone else’s account or admin features. Example: changing a URL to view another user’s data. <br>
   Attacks that fall under this category include:
   - Directory Traversal
   - Hidden Field Manipulation

3. **A02 – Cryptographic Failures** →
   This is when sensitive data (like passwords or credit card info) isn’t properly encrypted. If it's sent in plain text or uses weak encryption, attackers can steal it. <br>
   Attacks that fall under this category include:
   - Cookie Snooping
   - RC4 NOMORE Attack
   - Same-Site Attack
   - Pass-the-Cookie Attack

5. **A03 – Injection** →
   When an application accepts untrusted input and runs it as code. Attackers can use this to run malicious commands. Example: SQL Injection lets them steal or change data. <br>
    Attacks that fall under this category include:
   - SQL Injection
   - Command Injection
   - LDAP Injection
   - Cross-Site Scripting (XXS)
   - Buffer Overflow

6. **A04 – Insecure Design** →
   Flaws in how an app is planned or designed, like missing security checks. It leads to attacks like CAPTCHA bypass or session hijacking. <br>
   Attacks that fall under this category include:
   - Business Logic Bypass Attack
   - Web-based Timing Attacks
   - CAPTCHA Attacks
   - Platform Exploits

8. **A05 – Security Misconfiguration** →
   This happens when systems are not properly set up. Using default passwords, showing error messages with too much info, or not updating software can make apps easy to hack. <br>
   Attacks that fall under this category include:
   - XML External Entity (XXE) Attack
   - Unvalidated Redirects and Forwards
   - Directory Traversal
   - Hidden Field Manipulation

10. **A06 – Vulnerable and Outdated Components** →
   Using old software parts (like plugins, libraries, etc.) with known bugs lets attackers exploit them. These weak parts often run with full access and can lead to data theft. <br>
   Attacks that fall under this category include:
      - Platform Exploits
      - Magecart Attack
      - Buffer Overflow

12. **A07 – Identification and Authentication Failures** →
   Weak login systems can be bypassed. Attackers may guess passwords or hijack sessions if tokens and credentials aren't protected properly. <br>
   Attacks that fall under this category include:
      - Cross-Site Request Forgery
      - Cookie/Session Poisoning
      - Cookie Snooping

14. **A08 – Software and Data Integrity Failures** →
   If software updates or data are not verified properly, attackers can inject malware. For example, insecure auto-update features or untrusted plugins. <br>
   Attacks that fall under this category include:
      - Insecure Deserialization
      - Unvalidated Redirects and Forwards
      - Watering Hole Attack
      - Denial-of-Service (DoS)
      - Buffer Overflow
      - Web Service Attacks
      - Platform Exploits
      - Magecart Attack

16. **A09 – Security Logging and Monitoring Failures** →
   If the app doesn’t log activities or alert about unusual actions (like failed logins), attacks can go unnoticed. This gives hackers more time to do damage. <br>
   Attacks that fall under this category include:
      - Web Service Attacks

18. **A10 – Server-Side Request Forgery (SSRF)** →
    When an app fetches a URL without checking if it's safe, attackers can use it to access internal systems behind firewalls or steal sensitive info. <br>
    Attacks that fall under this category include:
       - Injecting an SSRF Payload
       - Cross-Site Port Attack (XSPA)
       - DNS Rebinding Attack
       - H2C Smuggling Attack
