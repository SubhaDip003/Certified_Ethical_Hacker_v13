# 🔎 Footprint Web Infrastructure
Web infrastructure footprinting is the process of collecting detailed information about how a web application is built and connected to other systems. This is usually the first step attackers take to understand the technologies used, how everything is linked, and where weaknesses might exist. By studying the web infrastructure, attackers can find potential entry points to exploit and gain unauthorized access to the application.

Footprinting the web infrastructure allows an attacker to engage in the following tasks: 

1. **Server discovery** → Attackers try to find out which physical servers are hosting the web application using tools like Whois, DNS lookups, and port scanning.

2. **Service discovery** → They identify which services (like HTTP, FTP, SSH) are running on the server to find potential entry points for attacks.

3. **Server identification** → Attackers perform banner grabbing to learn the web server type and version, and gather details like server location, internal IPs, and protocol support.

4. **WAF and proxy detection** → They check if the web application uses a firewall or proxy, helping them understand what security is in place.

5. **Hidden content discovery** → Attackers search for hidden pages or functionalities not directly visible to users, which may reveal sensitive information.

6. **Load balancers detection** → They try to detect load balancers and uncover real IPs of backend servers that may be exposed online.

7. **Web app technologies detection** → They identify the technologies used in building the web app (e.g., PHP, ASP.NET, WordPress) to find specific vulnerabilities.

8. **WebSocket enumeration** → Attackers detect active WebSocket servers that might be used for real-time communication and could be vulnerable.

## Server Discovery
To footprint a web infrastructure, first, you need to discover active Internet servers. Three techniques, namely Whois lookup, DNS interrogation, and port scanning, help in discovering the active servers and their associated information. 

### Whois Lookup 
Whois lookup tools allow you to gather information about a domain with the help of DNS and Whois queries. They provide information about the IP address of the web server and DNS names. These tools produce results in the form of an HTML report. 

Use the following tools to perform Whois lookup: 
- **Netcraft** [https://www.netcraft.com]
- **Whois Lookup** [https://whois.domaintools.com]
- **Batch IP Converter** [http://www.sabsoft.com]
- **Whois Domain Lookup** [https://www.whois.com]
- **Whois** [https://webwhois.verisign.com]
- **Whois Lookup** [https://mxtoolbox.com]

### DNS Interrogation
DNS interrogation is a technique used to gather information about an organization’s network by querying its Domain Name System (DNS), which links IP addresses to hostnames. If the DNS is not properly secured, attackers can exploit it to discover server locations, types, and other sensitive details. This information can then be used to plan and launch targeted cyberattacks.

Use the following tools to perform DNS interrogation:
- **DNSRecon** [https://www.kali.org/tools/dnsrecon/]
- **DNS Records** [https://www.nslookup.io]
- **Domain Dossier** [https://centralops.net]
- **DNSdumpster.com** [https://dnsdumpster.com]
