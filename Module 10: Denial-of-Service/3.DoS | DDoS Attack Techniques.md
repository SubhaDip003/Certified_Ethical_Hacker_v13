# 💥 DoS/DDoS Attack Techniques
Attackers implement various techniques to launch denial-of-service (DoS)/distributed denial-of-service (DDoS) attacks on target computers or networks. This section discusses the basic categories of DoS/DDoS attack vectors, various attack techniques, and various DoS/DDoS attack tools used to take over a single or multiple network system to exhaust their computing resources or render them unavailable to their intended users.

---
## Basic Categories of DoS/DDoS Attack Vectors 
DDoS attacks mainly aim to diminish the network bandwidth by exhausting network, application, or service resources, thereby restricting legitimate users from accessing system or network resources. In general, DoS/DDoS attack vectors are categorized as follows: 

### Volumetric Attacks 
**Volumetric Attacks** are a type of **DDoS attack** that flood a target’s network or service with massive traffic, using up all available bandwidth and blocking real users from accessing it. The attack strength is measured in **bits per second (bps)**.

These attacks often abuse protocols like **NTP**, **DNS**, and **SSDP**, which don’t have built-in limits to prevent overloading. Since one machine alone can't generate enough traffic, attackers use multiple compromised systems to send huge volumes of data at once. This overwhelms network devices like routers and switches, causing service disruption and traffic congestion across the network.

There are two types of bandwidth depletion attacks: 
- In a flood attack, zombies send large volumes of traffic to the victim’s systems to exhaust the bandwidth of these systems.
- In an amplification attack, the attacker or zombies transfer messages to a broadcast IP address. This method amplifies malicious traffic that consumes the bandwidth of the victim’s systems.

Attackers use botnets and perform DDoS attacks by flooding the network. The entire bandwidth is used up by attackers, and no bandwidth remains for legitimate use. The following are examples for volumetric attack techniques:
```
- User Datagram Protocol (UDP) flood attack			- NTP amplification attack
- Internet Control Message Protocol (ICMP) flood attack		- Spoofed IP packet flood attack
- Ping of Death (PoD) attack					- Malformed IP packet flood attack
- Smurf attack							- Zero-day attack
- Pulse wave attack
```
### Protocol Attacks
**Protocol Attacks** are a type of DDoS attack that don’t just consume bandwidth—they target and exhaust system resources like **connection state tables** on devices such as **firewalls, load balancers, and servers**. These tables keep track of active connections, and when they're full, no new connections can be made.

Attackers send a massive number of fake requests to fill up these tables, causing real users to be blocked. The attack is measured in **packets per second (pps)** or **connections per second (cps)** and can overwhelm even high-capacity systems.

The following are examples for protocol attack techniques:
```
- Synchronize (SYN) flood attack		- TCP SACK panic attack	
- Fragmentation attack				- RST attack
- Spoofed session flood attack			- TCP state exhaustion attack
- Acknowledgement (ACK) flood attack		- TCP connection flood attack
- SYN-ACK flood attack				- ACK and PUSH ACK flood attack
```

### Application Layer Attacks
**Application Layer Attacks** target the software or application itself rather than the network. Attackers exploit bugs or weak points in the code to overload the application with fake requests, making it unavailable to real users. These attacks don’t need much traffic, so even one or two machines can cause serious damage.

Attackers might flood websites with fake user activity, send too many invalid login attempts, or use malicious SQL queries to crash databases. One common method is **buffer overflow**, where too much data is sent to an app, causing it to crash or run harmful code.

These attacks are hard to detect because the traffic looks normal. They’re measured in **requests per second (rps)** and can cause service outages, financial loss, and reputational damage. However, if detected, they’re easier to trace back to the source compared to other DDoS types.

The following are examples for application layer attack techniques:
```
- Hypertext Transfer Protocol (HTTP) flood attack
- Slowloris attack
- UDP application layer flood attack
- DDoS extortion attack
```

## DoS/DDoS Attack Techniques 
Next, the following DoS/DDoS attack techniques will be discussed: 

### UDP Flood Attack
**UDP Flood Attack** is a type of denial-of-service (DoS) attack where the attacker sends a huge number of fake UDP packets to random ports on a target server using different source IPs. The server tries to respond to each request, but finds no real service running on those ports, so it replies with error messages. This process wastes system resources and internet bandwidth, eventually making the network slow or completely unavailable for real users.

### ICMP Flood Attack 
**ICMP Flood Attack** is a type of Denial-of-Service (DoS) attack where attackers send a massive number of ICMP echo request packets (pings) to a target system. These requests force the system to reply, overloading its network and making it unable to respond to real users.

This attack can be done directly or through reflection networks. To prevent it, network devices can be configured with a **threshold limit** (usually 1000 packets per second). If this limit is crossed, the system blocks further ICMP requests from that source for the rest of the current second and the next one, helping to protect against the flood.

### Ping of Death Attack 
**Ping of Death (PoD)** is a type of attack where the attacker sends an oversized or malformed ping packet—larger than the allowed **65,535 bytes**—to a target system. When the target tries to reassemble this large packet, it may crash, freeze, or become unstable. The attacker only needs the target's **IP address**, and their identity can be easily hidden (spoofed). This makes PoD a simple but disruptive denial-of-service attack.

### Smurf Attack
In a Smurf attack, the attacker spoofs the source IP address with the victim’s IP address and sends a large number of ICMP ECHO request packets to an IP broadcast network. This causes all the hosts on the broadcast network to respond to the received ICMP ECHO requests. These responses are sent to the victim’s machine because the IP address was spoofed by the attacker, causing significant traffic to the victim’s machine and ultimately making it crash.

### Pulse Wave DDoS Attack 
**Pulse Wave DDoS Attack** is a powerful form of DDoS attack where attackers send massive bursts of traffic (called pulses) at regular intervals instead of continuous flow. These high-speed pulses, often **300 Gbps or more**, hit the target every **10 minutes** and can last for **hours or even days**. Each pulse overloads the target’s network, making recovery extremely hard or sometimes impossible. This method is designed to keep services offline repeatedly and cause maximum disruption.
