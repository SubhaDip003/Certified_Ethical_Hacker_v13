# ♻️ Maintaining Access
Maintaining access means attackers, after breaking into a system, secretly install tools like keyloggers or spyware to stay hidden and keep control, so they can continue stealing data or attack other systems later.

----
## Executing Applications
Executing Applications is the stage where attackers, after gaining admin rights, run malicious programs on the victim's system to steal data, control the system, or install backdoors—often done locally or remotely to fully “own” the machine.

The malicious programs attackers execute on target systems can be:
- **Backdoors:** Backdoors are hidden ways to access a system without permission. Hackers use them to bypass security, steal data, or control the system later.
  
- **Crackers:** Crackers are tools used to break passwords, software licenses, or encryption. They're often used to unlock paid software illegally or get into protected systems.

- **Keyloggers:** Keyloggers record every key you press on your keyboard. Hackers use them to steal login details, credit card numbers, and other sensitive information.
  
- **Spyware:** Spyware secretly monitors what you do on your computer. It can take screenshots, track browsing activity, or send data to hackers. Attackers often install backdoors along with spyware to keep accessing the victim’s system later.

---
### Remote Code Execution Techniques 
Remote Code Execution (RCE) is a hacking technique where an attacker runs malicious code on another computer or server over a network. This usually happens after the attacker has already found a way into the system and wants to take control of other connected machines.

1. **Exploitation for Client Execution** - Attackers exploit software vulnerabilities to run code and take control of remote systems. Different types of exploitations for client execution are as follows:
   
      * **Web Browser Exploits** - Hackers use fake or malicious websites (like spear-phishing links). When a user visits the page, the browser gets compromised without the user even clicking anything.
      * **Office Application Exploits** - Hackers send emails with infected Word, Excel, or PDF files. The attack only works when the user opens the malicious file.
      * **Third-Party App Exploits** - Vulnerabilities in popular apps like Adobe Reader or Flash can be used to break into a system when the app is used or opened.

2. **Service Execution** - Attackers create or modify Windows services to run harmful code. This helps them stay hidden and keep control over the system.

3. **Windows Management Instrumentation (WMI)** - WMI is a Windows tool used to manage systems. Hackers use it to:

      * Remotely control systems
      * Collect data from systems
      * Run malicious code
      * Move from one computer to another inside a network

They often use ports like **135**, **5985**, and **5986** to connect remotely.

4. **Windows Remote Management (WinRM)** - WinRM is used by admins to manage systems remotely. Attackers abuse it to:

      * Run malicious files on remote systems
      * Change system settings or services
      * Move across systems in a network (lateral movement)

### 🔑Keylogger
Keystroke loggers (or keyloggers) are tools that secretly record everything a user types on a keyboard. They help attackers collect personal data like passwords, messages, emails, and more. These can be either hardware-based or software-based, depending on how they are used.

![keylogger](https://github.com/user-attachments/assets/a09b7440-717f-4efe-86d1-6cd64a760c20)

### Types of Keystroke Loggers
![Keyloggertypes](https://github.com/user-attachments/assets/226ff09f-d358-4469-b03d-27b8fee73edd)

There are two types of keystroke loggers: hardware key loggers and software key loggers. Both types help attackers to record all keystrokes entered on the target system. 

1. **Hardware Keystroke Loggers:** These are physical devices placed between the keyboard and the computer. They don’t need any software to work and are hard to detect through antivirus tools.

   * **PC/BIOS Embedded** - Hackers change the computer’s BIOS (startup software) to record keyboard inputs. It needs admin-level access and is hard to detect.
   
   * **Keylogger Keyboard:** - A keyboard with a special circuit built inside that stores all keystrokes. It saves data in its own memory, and it doesn't rely on the operating system, so software can't detect it.
  
   * **External Keylogger:** - A small device plugged between the keyboard and computer port. It captures every key press and stores it. It's easy to install but can be seen physically. There are four types of external keyloggers:
     
     - **PS/2 and USB Keylogger:** A plug-and-play device that saves all typed information without needing software or drivers.
       
     - **Acoustic/CAM Keylogger:** Uses sound or cameras to track keystrokes. For example, sound from key presses is turned into data.
       
     - **Bluetooth Keylogger:** Installed once physically, then it sends live keystroke data wirelessly via Bluetooth.
       
     - **Wi-Fi Keylogger:** Works like other keyloggers but uses Wi-Fi to send stolen data to attackers over the internet.

2. **Software Keystroke Loggers:** These are programs installed on the victim’s system through emails or the internet. They store keystrokes in hidden files and send them to the attacker, often through email. They can collect more data than hardware loggers and are harder to spot without proper tools.

    * **Application Keylogger** - Tracks everything typed in apps like chat, email, and web browsers. It works silently in the background.
      
    * **Kernel/Rootkit/Device Driver Keylogger** This is an advanced and hidden type that works deep inside the system (kernel level).
      
      * **Kernel Keylogger:** Hard to create and detect. It works as a fake keyboard driver and captures all typed keys.
      
      * **Rootkit-Based Keylogger:** A hidden Windows driver that records keystrokes. It avoids detection even by antivirus.
      
      * **Device Driver Keylogger:** Replaces the normal keyboard driver with a fake one that logs keystrokes secretly and sends them to the attacker.

   * **Hypervisor-Based Keylogger** - This works under the operating system, in a fake virtual machine (malware hypervisor). It quietly controls the system and records keystrokes without being noticed.
   
   * **Form-Grabbing Keylogger** - Instead of just logging keystrokes, this one captures what you type in online forms (like login forms), even if the site uses HTTPS (secure connection). It records the data when you press the submit button.
   
   * **JavaScript-Based Keylogger** - Attackers add harmful JavaScript code to websites (using tricks like XSS or browser hijacking). The script watches and records keystrokes using browser events like onKeyDown() and onKeyUp().
   
   * **Memory-Injection Keylogger** - This type of keylogger modifies the computer’s memory to hook into the browser or system. It records keystrokes and can even bypass Windows security (UAC) to stay hidden.

### Remote Keylogger Attack Using Metasploit
In a remote keylogger attack using Metasploit, hackers first gain access to the victim’s computer. If they can’t directly view protected files or folders, they use Metasploit to install a keylogger on the target system. This keylogger then records all keyboard activity—like passwords typed—and sends it back to the attacker, helping them steal even the most securely stored data.

### Establishing a Keylogger Using Metasploit 
#### 🔐 **Short Explanation:**
Once an attacker gets access to a Windows machine using Metasploit (Meterpreter session), they:

1. List running processes.
2. Migrate to a stable process like `explorer.exe`.
3. Start keylogging with `keyscan_start`.
4. Collect keystrokes using `keyscan_dump`.
5. Stop the keylogger with `keyscan_stop`.

### 🧑‍💻 **Commands with Description:**

```bash
# Step 1: List all running processes
ps              # Shows process list with PIDs

# Step 2: Check your current process ID
getpid          # Shows current PID

# Step 3: Migrate to a stable process like explorer.exe (replace <PID> with actual explorer.exe PID)
migrate <PID>   # Migrates session to another process to avoid detection

# Step 4: Start the keylogger
keyscan_start   # Begins recording keystrokes

# Step 5: Dump captured keystrokes
keyscan_dump    # Displays recorded keystrokes

# Step 6: Stop the keylogger
keyscan_stop    # Stops recording keystrokes
```
> ✅ **Note:** Attackers can also use `post/windows/capture/lockout_keylogger` module to automate this.

### 👁️‍🗨️Spyware
**What is Spyware?**
Spyware is a hidden software that secretly monitors and records everything a user does on their computer—like keystrokes, emails, websites visited, and passwords—and sends that data to an attacker over the internet. It often comes hidden in free software and is hard to detect because it hides itself from security tools.

### Spyware Propagation
**Spyware propagation** means how spyware spreads or installs itself on a system **without the user's knowledge**. It often hides in free software (piggybacking), uses **drive-by downloads** when users click on infected websites, or pretends to be helpful tools like fake anti-spyware. Just by browsing or downloading something, a computer can get infected.

## Hiding Files 
After hacking a system, attackers hide their malicious files using tricks like rootkits, NTFS streams, or steganography. This helps them avoid detection by antivirus software and keep control of the system secretly for future access.

---
### 🦠Rootkits 
A rootkit is a dangerous type of malware that secretly gives attackers full control (root access) over a system. It hides itself deep in the operating system, making it hard to detect. Once installed, it can let hackers delete files, install other malware, spy on users, or cover up their tracks by changing system files and processes. It often uses OS vulnerabilities and can even hide or change file attributes to stay invisible.

### Types of Rootkits
Rootkits employ a range of techniques to gain control of a system. The type of rootkit influences the choice of attack vectors. There are six types of rootkits available:

1. **Hypervisor-Level Rootkit**: Runs under the operating system using virtualization to secretly control everything the system does.

2. **Hardware/Firmware Rootkit**: Hides in hardware like BIOS or hard drives, making it hard to detect and remove.

3. **Kernel-Level Rootkit**: Hides deep inside the operating system with full control, making it powerful and dangerous.

4. **Boot-Loader Rootkit**: Loads before the OS starts by modifying the boot process, allowing it to bypass security early.

5. **Application-Level Rootkit**: Replaces or modifies software apps to spy or control without being noticed.

6. **Library-Level Rootkit**: Alters system files or libraries to hide malicious activity from users and tools.

7. **Memory Rootkit**: Stays only in RAM, leaving no trace on disk, so it's hard to detect and vanishes on reboot.

### How a Rootkit Works
A rootkit works by secretly changing how your system functions so it can stay hidden. It does this by redirecting system calls to its own code (called system hooking) or by modifying core system files like `kernel32.dll` or `ntdll.dll` to run its code first (inline hooking). Some advanced rootkits use a technique called Direct Kernel Object Manipulation (DKOM) to directly change hidden parts of the system's memory. This allows them to hide processes, open backdoors, change user privileges, and fool system tools like the Windows event viewer, all without being detected.

### Detecting Rootkits 
Detecting rootkits is tricky because they hide themselves deeply in the system. To find them, security tools use different methods like checking for known rootkit patterns (signature-based), spotting suspicious behavior (heuristic-based), comparing system files with clean versions (integrity-based), checking what the system shows vs. what’s really running (cross-view), and analyzing how programs run in real-time (runtime path profiling). These techniques help uncover rootkits even when they try to stay hidden.

To stop rootkits, it’s important to detect unusual activity early—like strange network traffic or system changes—since rootkits need admin access. Use log analysis and file integrity tools like `MD5sum` to catch changes in system files. For strong protection, regularly back up important data and reinstall your system from a trusted source if infection is suspected.


1. **Integrity-Based Detection** checks for changes in files by comparing with a clean snapshot.
2. **Signature-Based Detection** looks for known malicious patterns like a fingerprint.
3. **Heuristic/Behavior-Based Detection** spots abnormal system behavior.
4. **Runtime Execution Path Profiling** watches for strange code added to system routines.
5. **Cross-View Detection** compares real system data with what the OS reports to catch hidden changes.
6. **Alternative Trusted Medium** boots the system from an external drive to scan for rootkits safely.
7. **Memory Dump Analysis** captures and analyzes RAM to find active rootkits.
8. **Virtualization-Based Analysis** uses a hypervisor to monitor the system from outside and detect hidden threats.

### Steps for Detecting Rootkits
Detecting rootkits can be tough because attackers constantly create new ways to hide them from security tools. While there are many tools available to find rootkits, they don’t always work against advanced or updated ones. That’s why manual detection—though time-consuming and requiring skill—is often more reliable, as it helps spot hidden rootkits that automated tools might miss.

Manually examine the filesystem and registry of the system to detect rootkits.

#### 1. Steps to detect rootkits by examining the filesystem are as follows. 
```powershell
# Step-1: Inside the potentially infected system
# ---------------------------------------------
# List all hidden files in all folders and save output
C:\> dir /s /b /ah > infected_hidden.txt

# List all non-hidden files in all folders and save output
C:\> dir /s /b /a-h > infected_visible.txt


# Step-2: Boot from a clean USB (safe system)
# ---------------------------------------------
# Access the same drive (e.g., C:\ of infected system mounted as D:\ here)

# List hidden files from clean environment and save output
D:\> dir /s /b /ah > clean_hidden.txt

# List non-hidden files from clean environment and save output
D:\> dir /s /b /a-h > clean_visible.txt


# Step-3: Compare both results using WinMerge (GUI Tool)
# -------------------------------------------------------
# Open WinMerge → Compare:
#    "infected_hidden.txt" vs "clean_hidden.txt"
#    "infected_visible.txt" vs "clean_visible.txt"

# This will show files that are hidden on the infected system but visible from the outside.
# These are possible rootkit or ghostware files.
```

#### 2. Steps to detect rootkits by examining the registry are as follows.
```powershell
# Step 1: On infected system, open registry editor
regedit.exe
# -> This opens Windows Registry Editor.

# Step 2: Export important registry hives
# In regedit, export the following as .txt:
# HKEY_LOCAL_MACHINE\SOFTWARE
# HKEY_LOCAL_MACHINE\SYSTEM
# -> These contain key system and software settings.

# Step 3: Reboot system using a clean OS (e.g., from USB like WinPE)
# Boot from USB to avoid running the infected OS.

# Step 4: Again, open Registry Editor from clean OS
regedit.exe

# Step 5: Create a temporary key to mount the old registry
# In Registry Editor, select HKEY_LOCAL_MACHINE → File → Load Hive
# Navigate to:
# c:\windows\system32\config\software
# c:\windows\system32\config\system
# Load them into:
# HKEY_LOCAL_MACHINE\Temp

# Step 6: Export these loaded hives as .txt files
# -> This gives you clean-view copies of the suspect registry in text format.

# Step 7: Use a comparison tool (e.g., WinMerge) from the USB
WinMerge.exe
# -> Compare the two sets of exported registry files (infected vs clean).
# If malware/rootkits are hiding, the infected system's regedit might not show all entries,
# but WinPE's regedit + WinMerge will catch them.
```
> Note: There can be some false positives. In addition, this does not detect stealth software that hides in BIOS, video card EEPROM, bad disk sectors, alternate data streams (ADSs), etc.

### 💾NTFS Data Stream
NTFS filesystems use two data streams to store a file—one for file permissions and one for the actual content. NTFS files can hold extra hidden data called **Alternate Data Streams (ADS)**, which are attached to a file without changing its size or how it looks. These streams store extra info like file attributes or even hidden malicious code. Attackers use ADS to hide tools or malware inside regular-looking files, making them very hard to detect with normal methods like Windows Explorer or Command Prompt. Only advanced tools can find these hidden streams.

### How to Create NTFS Streams
```powershell
# Step 1: Create a hidden stream named 'lion.txt' inside 'myfile.txt'
c:\> notepad myfile.txt:lion.txt
# → Notepad opens. Type some data, then save and close. This creates the ADS.

# Step 2: Create another hidden stream named 'tiger.txt' in the same file
c:\> notepad myfile.txt:tiger.txt
# → Again, type some data, then save and close. This adds another ADS.

# Step 3: Check the visible file size of myfile.txt
c:\> dir myfile.txt
# → File size appears as 0 bytes because the data is in ADS, not the main file.

# Step 4: View or edit hidden stream data
c:\> notepad myfile.txt:lion.txt
# → Opens the hidden 'lion.txt' stream

c:\> notepad myfile.txt:tiger.txt
# → Opens the hidden 'tiger.txt' stream
```
> Note: Notepad is a stream-compliant application. You should not use alternate streams to store critical information.

### NTFS Stream Manipulation 
You can manipulate NTFS streams to hide a malicious file in other files, such as text files, by doing the following:
```powershell
# Step 1: Hide Trojan.exe inside Readme.txt using ADS
C:\> type C:\Trojan.exe > C:\Readme.txt:Trojan.exe
# → This command hides the contents of Trojan.exe inside Readme.txt as a hidden stream.

# Step 2: Create a shortcut (link) to the hidden Trojan.exe
C:\> mklink backdoor.exe Readme.txt:Trojan.exe
# → This creates a backdoor.exe link that points to the hidden Trojan.exe stream.

# Step 3: Run the hidden Trojan using the shortcut
C:\> backdoor.exe
# → This executes the hidden Trojan stored inside Readme.txt.

# (Optional) Step 4: View a hidden text stream using Notepad
C:\> notepad sample.txt:secret.txt
# → Opens a hidden text stream named secret.txt behind sample.txt using Notepad.
```

### 🖼️🔐Steganography
**Steganography (Simple Explanation):**
Steganography is the technique of secretly hiding data inside files like images, audio, video, or text so no one knows it's there. Unlike encryption, it hides the message itself. For example, attackers can hide malware like keyloggers in images, making them hard to detect. It's often used when encryption isn't enough or possible, making it a stealthy way to share hidden information.

### Classification of Steganography 
Based on its technique, steganography can be classified into two areas: technical and linguistic. In technical steganography, a message is hidden using scientific methods, whereas in linguistic steganography, it is hidden in a carrier, which is the medium used to communicate or transfer messages or files. This medium comprises of the hidden message, carrier, and steganography key. 

**The following diagram depicts the classification of steganography.** 

![image](https://github.com/user-attachments/assets/17e77181-fe2b-4e12-9724-5d6434c80f64)

### 1. Technical Steganography
It hides secret messages using physical or digital methods so others can’t detect them.
* **Invisible Ink:** Uses colorless liquids (like milk or onion juice) to write messages that appear only with heat or light.

* **Microdots:** Shrinks an entire message or image into a tiny dot (1mm), often invisible to the eye, used to avoid detection.

* **Computer-Based Methods:** Hides data in digital files like images, videos, audio, or network traffic. Some computer-based steganography as follows:

  - **Substitution:** Replaces tiny bits (like pixels) in media with secret data.
    
  - **Transform Domain:** Hides data in complex parts of media using image processing like compression or cropping.
  
  - **Spread Spectrum:** Hides data by spreading it across a wide range of frequencies, making it hard to detect.
  
  - **Statistical:** Alters statistics of a file when embedding a message so changes are detectable only if you know what to look for.
  
  - **Distortion:** Modifies the original file slightly to carry the message; the receiver compares with the original to extract it.
  
  - **Cover Generation:** Creates a new digital file designed to carry a hidden message from the start.

#### 2. Linguistic Steganography
