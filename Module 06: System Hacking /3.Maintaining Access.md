# ♻️ Maintaining Access
Maintaining access means attackers, after breaking into a system, secretly install tools like keyloggers or spyware to stay hidden and keep control, so they can continue stealing data or attack other systems later.

----
## Executing Applications
Executing Applications is the stage where attackers, after gaining admin rights, run malicious programs on the victim's system to steal data, control the system, or install backdoors—often done locally or remotely to fully “own” the machine.

The malicious programs attackers execute on target systems can be:
- **Backdoors:** Backdoors are hidden ways to access a system without permission. Hackers use them to bypass security, steal data, or control the system later.
  
- **Crackers:** Crackers are tools used to break passwords, software licenses, or encryption. They're often used to unlock paid software illegally or get into protected systems.

- **Keyloggers:** Keyloggers record every key you press on your keyboard. Hackers use them to steal login details, credit card numbers, and other sensitive information.
  
- **Spyware:** Spyware secretly monitors what you do on your computer. It can take screenshots, track browsing activity, or send data to hackers. Attackers often install backdoors along with spyware to keep accessing the victim’s system later.

### Remote Code Execution Techniques 
Remote Code Execution (RCE) is a hacking technique where an attacker runs malicious code on another computer or server over a network. This usually happens after the attacker has already found a way into the system and wants to take control of other connected machines.

1. **Exploitation for Client Execution** - Attackers exploit software vulnerabilities to run code and take control of remote systems. Different types of exploitations for client execution are as follows:
   
      * **Web Browser Exploits** - Hackers use fake or malicious websites (like spear-phishing links). When a user visits the page, the browser gets compromised without the user even clicking anything.
      * **Office Application Exploits** - Hackers send emails with infected Word, Excel, or PDF files. The attack only works when the user opens the malicious file.
      * **Third-Party App Exploits** - Vulnerabilities in popular apps like Adobe Reader or Flash can be used to break into a system when the app is used or opened.

2. **Service Execution** - Attackers create or modify Windows services to run harmful code. This helps them stay hidden and keep control over the system.

3. **Windows Management Instrumentation (WMI)** - WMI is a Windows tool used to manage systems. Hackers use it to:

      * Remotely control systems
      * Collect data from systems
      * Run malicious code
      * Move from one computer to another inside a network

They often use ports like **135**, **5985**, and **5986** to connect remotely.

4. **Windows Remote Management (WinRM)** - WinRM is used by admins to manage systems remotely. Attackers abuse it to:

      * Run malicious files on remote systems
      * Change system settings or services
      * Move across systems in a network (lateral movement)

## 🔑Keylogger
Keystroke loggers (or keyloggers) are tools that secretly record everything a user types on a keyboard. They help attackers collect personal data like passwords, messages, emails, and more. These can be either hardware-based or software-based, depending on how they are used.

### Types of Keystroke Loggers
![Keyloggertypes](https://github.com/user-attachments/assets/226ff09f-d358-4469-b03d-27b8fee73edd)

There are two types of keystroke loggers: hardware key loggers and software key loggers. Both types help attackers to record all keystrokes entered on the target system. 

1. **Hardware Keystroke Loggers:** These are physical devices placed between the keyboard and the computer. They don’t need any software to work and are hard to detect through antivirus tools.

   * **PC/BIOS Embedded** - Hackers change the computer’s BIOS (startup software) to record keyboard inputs. It needs admin-level access and is hard to detect.
   
   * **Keylogger Keyboard:** - A keyboard with a special circuit built inside that stores all keystrokes. It saves data in its own memory, and it doesn't rely on the operating system, so software can't detect it.
  
   * **External Keylogger:** - A small device plugged between the keyboard and computer port. It captures every key press and stores it. It's easy to install but can be seen physically. There are four types of external keyloggers:
     
     - **PS/2 and USB Keylogger:** A plug-and-play device that saves all typed information without needing software or drivers.
       
     - **Acoustic/CAM Keylogger:** Uses sound or cameras to track keystrokes. For example, sound from key presses is turned into data.
       
     - **Bluetooth Keylogger:** Installed once physically, then it sends live keystroke data wirelessly via Bluetooth.
       
     - **Wi-Fi Keylogger:** Works like other keyloggers but uses Wi-Fi to send stolen data to attackers over the internet.

2. **Software Keystroke Loggers:** These are programs installed on the victim’s system through emails or the internet. They store keystrokes in hidden files and send them to the attacker, often through email. They can collect more data than hardware loggers and are harder to spot without proper tools.

    * **Application Keylogger** - Tracks everything typed in apps like chat, email, and web browsers. It works silently in the background.
      
    * **Kernel/Rootkit/Device Driver Keylogger** This is an advanced and hidden type that works deep inside the system (kernel level).
      
      * **Kernel Keylogger:** Hard to create and detect. It works as a fake keyboard driver and captures all typed keys.
      
      * **Rootkit-Based Keylogger:** A hidden Windows driver that records keystrokes. It avoids detection even by antivirus.
      
      * **Device Driver Keylogger:** Replaces the normal keyboard driver with a fake one that logs keystrokes secretly and sends them to the attacker.

   * **Hypervisor-Based Keylogger** - This works under the operating system, in a fake virtual machine (malware hypervisor). It quietly controls the system and records keystrokes without being noticed.
   
   * **Form-Grabbing Keylogger** - Instead of just logging keystrokes, this one captures what you type in online forms (like login forms), even if the site uses HTTPS (secure connection). It records the data when you press the submit button.
   
   * **JavaScript-Based Keylogger** - Attackers add harmful JavaScript code to websites (using tricks like XSS or browser hijacking). The script watches and records keystrokes using browser events like onKeyDown() and onKeyUp().
   
   * **Memory-Injection Keylogger** - This type of keylogger modifies the computer’s memory to hook into the browser or system. It records keystrokes and can even bypass Windows security (UAC) to stay hidden.
