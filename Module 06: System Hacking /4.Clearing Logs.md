# 🧹 Clearing Logs
After hiding malicious files using techniques like steganography or NTFS streams to maintain access, attackers focus on **clearing logs** to erase evidence of their activity. This includes deleting event logs, command history, and other forensic traces from the system. By doing this, they cover their tracks and make it difficult for system administrators or investigators to detect the intrusion, helping them stay hidden for longer periods.

---
## Covering Tracks 
Covering tracks is the final and crucial step for attackers to avoid detection after compromising a system. It involves erasing evidence like security logs, failed login records, and unauthorized access traces. Attackers manipulate or delete log files such as SECEVENT.EVT, SYSEVENT.EVT, and APPEVENT.EVT to hide their activities. They may also disable future logging or use tools like rootkits to wipe or alter logs. To remain unnoticed, they restore file attributes like size and timestamps to their original state. While full log deletion often needs admin rights, attackers usually remove only specific entries to avoid suspicion. Detecting such tampering is tough but possible using cryptographic file hashes.

The attacker can manipulate the log files with the help of
- **SECEVENT.EVT** (security): failed logins, accessing files without privileges
- **SYSEVENT.EVT** (system): driver failure, things not operating correctly
- **APPEVENT.EVT** (applications)

### Techniques Used for Covering Tracks

* **Disabling Auditing**: Turns off system monitoring to avoid being tracked.
* **Clearing Logs**: Deletes activity logs to remove evidence.
* **Manipulating Logs**: Alters log entries to hide malicious actions.
* **Network Evasion**: Uses reverse shells, DNS/ICMP tunneling, or TCP tricks to hide traffic.
* **Hiding in OS**: Stores malicious files in **NTFS streams** to avoid detection.
* **Deleting Data Securely**: Uses tools like `cipher.exe` to permanently erase data.
* **Disabling Windows Features**: Shuts off functions like system restore, timestamps, hibernation to remove footprints.
* **Hiding Artifacts**: Blends malicious files within normal OS files or processes.

## Disabling Auditing: Auditpol
🔗Source: [https://learn.microsoft.com]

`Auditpol.exe` is a built-in Windows command-line tool used to manage and control security auditing policies on a system. It allows you to enable, disable, or configure auditing for specific system activities, such as logins, file access, or changes to security settings.

Attackers use this tool during a cyber attack, especially after gaining administrative access, to quietly turn off auditing and avoid detection. By disabling security logs, they can hide their malicious actions, such as stealing data or installing keyloggers. Once their tasks are complete, they may re-enable auditing to reduce suspicion and make it harder for security teams to notice any gaps or unusual activity.

In short, auditpol.exe helps attackers stay hidden by controlling what the system records during their activity.

After gaining access and establishing shell access with the target system, attackers use the following commands to enable/disable system auditing logs: 
```cmd
:: Step 1: View current audit policy settings
C:\>auditpol /get /category:*

:: Step 2: Disable auditing logs (hide attacker activity)
C:\>auditpol /set /category:"system","account logon" /success:disable /failure:disable

:: Step 3: (Optional) Re-enable auditing after actions are done
C:\>auditpol /set /category:"system","account logon" /success:enable /failure:enable
```
